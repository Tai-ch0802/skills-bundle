# tRPC 原則

> TypeScript monorepo 的端到端型別安全。

## 何時使用

```
✅ 完美適合：
├── TypeScript 前後端都有
├── Monorepo 結構
├── 內部工具
├── 快速開發
└── 型別安全至關重要

❌ 不適合：
├── 非 TypeScript 客戶端
├── 公開 API
├── 需要 REST 慣例
└── 多語言後端
```

## 主要優勢

```
為何選 tRPC：
├── 零 Schema 維護
├── 端到端型別推斷
├── IDE 跨堆疊自動完成
├── API 變更即時反映
└── 無需程式碼生成步驟
```

## 整合模式

```
常見設定：
├── Next.js + tRPC（最常見）
├── Monorepo 共享型別
├── Remix + tRPC
└── 任何 TS 前端 + 後端
```
