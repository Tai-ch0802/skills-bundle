# 速率限制原則

> 保護你的 API 免受濫用與過載。

## 為何需要速率限制

```
防禦：
├── 暴力攻擊
├── 資源耗盡
├── 成本超支（若按使用量計費）
└── 不公平使用
```

## 策略選擇

| 類型 | 方式 | 適用時機 |
|------|------|----------|
| **令牌桶** | 允許突發，隨時間補充 | 大多數 API |
| **滑動視窗** | 平滑分配 | 嚴格限制 |
| **固定視窗** | 每個視窗簡單計數 | 基本需求 |

## 回應標頭

```
在標頭中包含：
├── X-RateLimit-Limit（最大請求數）
├── X-RateLimit-Remaining（剩餘請求數）
├── X-RateLimit-Reset（限制重置時間）
└── 超過時回傳 429
```
