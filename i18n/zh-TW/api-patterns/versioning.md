# 版本控制策略

> 從第一天就規劃 API 演進。

## 決策因素

| 策略 | 實作 | 權衡 |
|------|------|------|
| **URI** | /v1/users | 清晰、易快取 |
| **標頭** | Accept-Version: 1 | 更乾淨的 URL、較難發現 |
| **查詢** | ?version=1 | 容易加入、雜亂 |
| **無版本** | 謹慎演進 | 內部最佳、公開有風險 |

## 版本控制哲學

```
考慮：
├── 公開 API？→ URI 中加版本
├── 僅限內部？ → 可能不需要版本控制
├── GraphQL？ → 通常不分版本（演進 Schema）
├── tRPC？ → 型別強制相容性
```
