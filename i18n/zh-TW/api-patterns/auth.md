# 驗證模式

> 根據使用案例選擇驗證模式。

## 選擇指南

| 模式 | 最適合 |
|------|--------|
| **JWT** | 無狀態、微服務 |
| **Session** | 傳統 Web、簡單場景 |
| **OAuth 2.0** | 第三方整合 |
| **API Keys** | 伺服器對伺服器、公開 API |
| **Passkey** | 現代無密碼驗證（2025+）|

## JWT 原則

```
重要：
├── 始終驗證簽名
├── 檢查過期時間
├── 包含最少的聲明
├── 使用短期效期 + 刷新令牌
└── 永遠不要在 JWT 中儲存敏感資料
```
