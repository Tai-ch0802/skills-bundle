# 情境發現

> 在建議任何架構之前，先收集情境。

## 問題層級（先詢問使用者）

1. **規模**
   - 多少使用者？（10、1K、100K、1M+）
   - 資料量？（MB、GB、TB）
   - 交易頻率？（每秒/每分鐘）

2. **團隊**
   - 單人開發還是團隊？
   - 團隊規模和專長？
   - 分散式還是同地辦公？

3. **時程**
   - MVP/原型還是長期產品？
   - 上市時間壓力？

4. **領域**
   - CRUD 密集還是業務邏輯複雜？
   - 即時需求？
   - 合規/法規要求？

5. **限制**
   - 預算限制？
   - 需要整合的遺留系統？
   - 技術堆疊偏好？

## 專案分類矩陣

```
                    MVP              SaaS           企業級
┌─────────────────────────────────────────────────────────────┐
│ 規模         │ <1K           │ 1K-100K      │ 100K+        │
│ 團隊         │ 單人          │ 2-10         │ 10+          │
│ 時程         │ 快速（數週）  │ 中等（數月） │ 長期（數年） │
│ 架構         │ 簡單          │ 模組化       │ 分散式       │
│ 模式         │ 最少          │ 選擇性       │ 全面         │
│ 範例         │ Next.js API   │ NestJS       │ 微服務       │
└─────────────────────────────────────────────────────────────┘
```
