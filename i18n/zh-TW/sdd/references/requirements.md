## 快速檢查清單

撰寫 `PRD_spec.md` 時，確保包含這些核心章節：

- [ ] **標頭**：版本、狀態、作者、最後更新
- [ ] **問題陳述**：問題描述和背景
- [ ] **使用者故事**：使用 US-XX 格式編號
- [ ] **功能需求**：使用 FR-XX 格式編號（EARS 語法）
- [ ] **驗收條件**：每個 FR 必須有對應的 AC（Given-When-Then）
- [ ] **範圍外項目**：明確定義不包含的內容

---

## 驗收條件範例

```gherkin
# FR-01 驗收條件
Given 使用者已登入系統
And 書籤數量大於 0
When 使用者點擊「匯出」按鈕
Then 系統須產生一個 JSON 檔案
And 檔案包含所有書籤資料
```

---

## EARS 語法快速參考

| 模式 | 格式 | 範例 |
|------|------|------|
| **普遍** | 系統須 [回應]。 | 系統須顯示載入指示器。 |
| **事件驅動** | 當 [觸發條件]，系統須 [回應]。 | 當使用者點擊儲存，系統須持久化資料。 |
| **狀態驅動** | 當處於 [狀態]，系統須 [回應]。 | 當離線時，系統須佇列請求。 |
| **可選** | 若 [功能] 啟用，系統須 [回應]。 | 若深色模式啟用，系統須使用深色配色。 |
| **防禦** | 若 [條件]，則系統須 [回應]。 | 若輸入無效，則系統須顯示錯誤。 |

## 版本控制

PRD 進入**已凍結**狀態後，任何變更需要透過變更請求流程：

1. 建立新版本（例如 v1.0 → v1.1）
2. 在修訂歷史中記錄變更
3. 重新取得審核者核准
