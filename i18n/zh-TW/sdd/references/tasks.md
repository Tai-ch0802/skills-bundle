## 實作前置條件

> **⚠️ 重要**：只有在 PRD 和 SA 都進入**已核准**或**已凍結**狀態後，才能開始實作階段。

確認以下項目：

- [ ] PRD 已核准（狀態：已核准/已凍結）
- [ ] SA 已核准（狀態：已核准/已凍結）
- [ ] 需求追溯已完成
- [ ] 測試影響分析已完成

## 驗證流程

實作完成後，執行以下驗證：

1. **對照 PRD 驗收條件**
   - 對每個 AC 執行 Given-When-Then 驗證
   - 記錄驗證結果

2. **執行測試**
   ```bash
   npm test
   ```

3. **建立 Pull Request**
   - 遵循團隊的 commit message 規範
   - 遵循團隊的 PR 審核流程
   - 執行任何 PR 驗證腳本

---

## 規格變更處理

如果在實作過程中發現規格需要調整：

1. **停止編碼**
2. **更新 SA/PRD**（建立新版本）
3. **重新取得核准**
4. **繼續實作**

> 這是 SDD 的核心原則：規格驅動程式碼，而非程式碼驅動規格。
