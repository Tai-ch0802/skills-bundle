# 決策樹與情境範本

> 基於情境的設計「思考」，而非固定方案。
> **這些是決策「指引」，而非複製貼上的範本。**
> **關於 UX 心理學原則 (希克定律, 費茲定律等)，請參閱：** [ux-psychology.md](ux-psychology.md)

---

## ⚠️ 如何使用此檔案

此檔案旨在幫助你「思考」決策，而非機械複製。

- 決策樹 → 幫助你思考各種選項
- 範本 → 展示「結構」與「原則」，而非確切數值
- **始終在應用前詢問使用者偏好**
- **根據情境生成全新的色譜**，不要複製十六進位色碼
- **應用來自 ux-psychology.md 的 UX 定律**來驗證你的決策

## 2. 受眾決策樹

### 你的目標使用者是誰？

```
目標受眾
      │
      ├── Z 世代 (18-25 歲)
      │   ├── 色彩：大膽、鮮艷、意想不到的組合
      │   ├── 排版：巨大、具有表現力、可變字體
      │   ├── 佈局：行動優先、垂直流、易於消化的片段
      │   ├── 特效：動效、遊戲化、互動性強
      │   └── 策略：真實感、快節奏、避免企業化感
      │
      ├── 千禧世代 (26-41 歲)
      │   ├── 色彩：柔和、大地色系、精緻
      │   ├── 排版：簡潔、易讀、功能性強
      │   ├── 佈局：響應式、基於卡片、有條理
      │   ├── 特效：細微、僅在有目的時使用
      │   └── 策略：價值驅動、透明、永續發展
      │
      ├── X 世代 (42-57 歲)
      │   ├── 色彩：專業、值得信賴、保守
      │   ├── 排版：熟悉、清晰、務實
      │   ├── 佈局：傳統層級、可預測
      │   ├── 特效：極簡、功能性回饋
      │   └── 策略：直接、高效、可靠
      │
      ├── 嬰兒潮世代 (58 歲以上)
      │   ├── 色彩：高對比、簡單、清晰
      │   ├── 排版：大尺寸、高易讀性
      │   ├── 佈局：簡單、線性、不雜亂
      │   ├── 特效：無或非常少
      │   └── 策略：清晰、詳細、值得信賴
      │
      └── B2B / 企業級
          ├── 色彩：專業配色、色彩內斂
          ├── 排版：簡潔、數據親和、易於掃描
          ├── 佈局：基於網格、有序、高效
          ├── 特效：專業感、細微
          └── 策略：專家感、解決方案導向、ROI 驅動
```

## 3. 色彩選擇決策樹

### 不要使用固定的十六進位碼，請遵循以下過程：

```
你希望引發什麼樣的情緒/動作？
            │
            ├── 信任與安全 (Trust & Security)
            │   └── 考慮：藍色系列、專業中性色
            │       → 詢問使用者具體的色調偏好
            │
            ├── 成長與健康 (Growth & Health)
            │   └── 考慮：綠色系列、自然色調
            │       → 詢問使用者是否側重於環保/自然/身心健康
            │
            ├── 緊迫感與行動 (Urgency & Action)
            │   └── 考慮：暖色系 (橘/紅) 作為「強調色」
            │       → 節制使用，詢問是否合適
            │
            ├── 奢華與進階 (Luxury & Premium)
            │   └── 考慮：深黑、金屬色、受限的配色方案
            │       → 詢問關於品牌的定位
            │
            ├── 創意與俏皮 (Creative & Playful)
            │   └── 考慮：多色彩、意想不到的組合
            │       → 詢問關於品牌個性
            │
            └── 冷靜與極簡 (Calm & Minimal)
                └── 考慮：中性色搭配單一強調色
                    → 詢問什麼樣的強調色符合品牌
```

### 執行過程：
1. 確定所需的情緒
2. 縮小到色彩「系列 (FAMILY)」
3. 在該系列內詢問使用者偏好
4. 利用 HSL 原則生成全新的色譜

## 4. 排版決策樹

```
內容類型是什麼？
          │
          ├── 數據密集型 (儀表板、SaaS)
          │   ├── 風格：無襯線體、清晰、緊湊
          │   ├── 比例：較緊湊的比率 (1.125-1.2)
          │   └── 優先級：可掃描性、資訊密度
          │
          ├── 評論/社論型 (部落格、雜誌)
          │   ├── 風格：襯線標題 + 無襯線內文效果佳
          │   ├── 比例：戲劇化的比率 (1.333+)
          │   └── 優先級：閱讀舒適度、層級結構
          │
          ├── 現代科技感 (新創、SaaS 行銷)
          │   ├── 風格：幾何或人文精神無襯線體
          │   ├── 比例：平衡的比率 (1.25)
          │   └── 優先級：現代感、清晰度
          │
          ├── 奢華風格 (時尚、進階精品)
          │   ├── 風格：優雅襯線體或極細無襯線體
          │   ├── 比例：戲劇化的比率 (1.5-1.618)
          │   └── 優先級：精緻感、留白
          │
          └── 俏皮風格 (兒童、遊戲、隨意)
              ├── 風格：圓潤、友好的字體
              ├── 比例：多變、富有表現力
              └── 優先級：趣味、易親近、易讀
```

### 選擇過程：
1. 確定內容類型
2. 選擇風格「方向」
3. 詢問使用者是否有品牌字體
4. 選擇與方向匹配的字體

## 5. 電子商務指引 (E-commerce) {#e-commerce}

### 關鍵原則 (非強制規則)
- **信任第一：** 你將如何展示安全性？
- **行動導向：** CTA 放在哪裡？
- **可掃描性：** 使用者能快速比較嗎？

### 色彩思考：
```
電子商務網站通常需要：
├── 信任色 (通常是藍色系) → 詢問偏好
├── 潔淨的背景 (白色/中性色) → 取決於品牌
├── 行動強調色 (用於 CTA、促銷) → 取決於急迫程度
├── 成功/錯誤語義色 → 標準慣例即可
└── 品牌整合 → 詢問現有品牌色
```

### 佈局原則：
```
┌────────────────────────────────────────────────────┐
│  標頭：品牌 + 搜尋 + 購物車                         │
│  (保持核心動作可見)                                │
├────────────────────────────────────────────────────┤
│  信任區域：為何信任此網站？                         │
│  (運送、退貨、安全資訊 — 若適用)                    │
├────────────────────────────────────────────────────┤
│  HERO：主要訊息或優惠                               │
│  (清晰的 CTA，單一焦點)                             │
├────────────────────────────────────────────────────┤
│  類別：易於導航                                     │
│  (視覺化、可篩選、易於掃描)                         │
├────────────────────────────────────────────────────┤
│  產品：易於比較                                     │
│  (價格、評分、快速動作可見)                         │
├────────────────────────────────────────────────────┤
│  社會證明：為何他人信任                             │
│  (評論、見證 — 若有可用資訊)                        │
├────────────────────────────────────────────────────┤
│  頁尾：所有細節資訊                                 │
│  (政策、聯絡方式、信任標章)                         │
└────────────────────────────────────────────────────┘
```

### 應應用的心理學：
- 希克定律：限制導航選項
- 費茲定律：適當調整 CTA 大小
- 社會證明：在相關處顯示
- 稀缺性：若使用，請確保真實誠實

## 6. SaaS 儀表板指引 (SaaS Dashboard) {#saas}

### 關鍵原則
- **功能第一：** 數據清晰度優於裝飾
- **冷靜的 UI：** 減輕認知負荷
- **一致性：** 可預測的模式

### 色彩思考：
```
儀表板通常需要：
├── 背景：淺色或深色 (詢問偏好)
├── 表面 (Surface)：與背景有輕微對比
├── 主要強調色：用於關鍵動作
├── 數據顏色：成功/警告/危險等語義色
└── 柔和色：用於次要資訊
```

### 佈局原則：
```
考慮這些模式 (非強制)：

選項 A：側邊欄 + 內容
├── 用於導航的固定側邊欄
└── 用於內容的主區域

選項 B：頂部導覽 + 內容
├── 水平導航
└── 更多的水平內容空間

選項 C：收納式 + 可展開
├── 僅圖示的側邊欄可展開
└── 最大化內容區域

→ 詢問使用者關於導航方式的偏好
```

### 應應用的心理學：
- 希克定律：對導航項目進行分組
- 米勒定律：將資訊塊狀化
- 認知負荷：留白、一致性

## 7. 著陸頁指引 (Landing Page) {#landing-page}

### 關鍵原則
- **Hero 為中心：** 第一印象最重要
- **單一焦點：** 一個主要的 CTA
- **情感連結：** 在銷售前先建立連結

### 色彩思考：
```
著陸頁通常需要：
├── 品牌主色：Hero 背景或強調色
├── 簡潔的次要色：頁面大部分區域
├── CTA 顏色：從所有內容中脫穎而出
├── 輔助色：用於區塊、見證
└── 首先詢問品牌色！
```

### 結構原則：
```
┌────────────────────────────────────────────────────┐
│  導覽列：極簡，CTA 可見                            │
├────────────────────────────────────────────────────┤
│  HERO：掛鉤 (Hook) + 價值 + CTA                   │
│  (最重要的區塊，影響力最大)                         │
├────────────────────────────────────────────────────┤
│  問題：他們有什麼痛點？                             │
├────────────────────────────────────────────────────┤
│  解決方案：你如何解決它                             │
├────────────────────────────────────────────────────┤
│  證明：為何相信你？                                 │
│  (見證、標誌、數據統計)                             │
├────────────────────────────────────────────────────┤
│  如何做：簡單解釋過程                               │
├────────────────────────────────────────────────────┤
│  定價：若適用                                       │
├────────────────────────────────────────────────────┤
│  常見問題 (FAQ)：解決疑慮                           │
├────────────────────────────────────────────────────┤
│  結尾 CTA：重複主要動作                             │
└────────────────────────────────────────────────────┘
```

## 8. 作品集指引 (Portfolio) {#portfolio}

### 關鍵原則
- **個性化：** 展現你是誰
- **以作為重點：** 讓專案自己說話
- **令人難忘：** 從範本中脫穎而出

### 色彩思考：
```
作品集非常個性化 — 選項眾多：
├── 極簡：中性色 + 一個簽名式強調色
├── 大膽：意想不到的顏色組合
├── 深沉：憂鬱、藝術感
├── 明亮：簡潔、專業感
└── 詢問關於個人品牌識別！
```

### 結構原則：
```
┌────────────────────────────────────────────────────┐
│  導覽列：體現你的個性                               │
├────────────────────────────────────────────────────┤
│  介紹 (Intro)：你是誰，你做什麼                      │
│  (使其難忘，而非通俗)                               │
├────────────────────────────────────────────────────┤
│  作品 (Work)：精選專案                              │
│  (大型、視覺化、具互動性)                           │
├────────────────────────────────────────────────────┤
│  關於 (About)：個人故事                             │
│  (建立連結)                                         │
├────────────────────────────────────────────────────┤
│  聯絡 (Contact)：易於觸及                           │
│  (清晰、直接)                                       │
└────────────────────────────────────────────────────┘
```

## 9. 設計前檢查清單

### 開始「任何」設計之前

- [ ] **受眾已定義嗎？** (到底是誰)
- [ ] **主要目標已確認嗎？** (什麼動作)
- [ ] **已知約束條件嗎？** (時間、品牌、技術)
- [ ] **內容是否準備就緒？** (或需要佔位符)
- [ ] **詢問使用者偏好嗎？** (顏色、風格、佈局)

### 選擇顏色之前

- [ ] **詢問過使用者偏好嗎？**
- [ ] **考慮過情境嗎？** (行業、情緒)
- [ ] **這與你的預設偏好不同吧？**
- [ ] **檢查過無障礙性嗎？**

### 確定佈局之前

- [ ] **層級是否清晰？**
- [ ] **主要 CTA 是否明顯？**
- [ ] **考慮過行動裝置嗎？**
- [ ] **內容是否符合結構？**

## 10. 複雜度估算

### 小型專案 (數小時)
```
簡單著陸頁
小型作品集
基本表單
單一組件
```
→ 策略：簡化決策，專注執行

### 中型專案 (數天)
```
多頁面網站
帶有模組的儀表板
電商品類頁面
複雜表單
```
→ 策略：建立 Token，自定義組件

### 大型專案 (數週)
```
完整的 SaaS 應用
電子商務平台
自定義設計系統
複雜的工作流程
```
→ 策略：完整設計系統、文件、測試
