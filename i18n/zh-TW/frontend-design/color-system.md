# 色彩系統參考指南

> 色彩理論原則、選色過程以及決策指引。
> **不要死記硬背十六進位碼 (Hex codes) — 學習如何「思考」色彩。**

---

## 1. 色彩理論基礎

### 色輪 (The Color Wheel)

```
                    黃色 (YELLOW)
                       │
            黃綠色     │    黃橘色
            (Yellow-   │    (Yellow-
            Green)     │    Orange)
               ╲       │       ╱
                ╲      │      ╱
      綠色 (GREEN) ──── ● ──── 橘色 (ORANGE)
                ╱      │      ╲
               ╱       │       ╲
            藍綠色     │    紅橘色
            (Blue-     │    (Red-
            Green)     │    Orange)
                       │
                     紅色 (RED)
                       │
                    紫色 (PURPLE)
                   ╱       ╲
              藍紫色        紅紫色
              (Blue-        (Red-
              Purple)       Purple)
                   ╲       ╱
                     藍色 (BLUE)
```

### 色彩關係

| 配色方案 | 如何建立 | 何時使用 |
|----------|----------|----------|
| **單色 (Monochromatic)** | 挑選一種色相，僅改變明度/飽和度 | 極簡、專業、具有凝聚力 |
| **相似色 (Analogous)** | 挑選色輪上相鄰的 2-3 個色相 | 和諧、冷靜、受自然啟發 |
| **互補色 (Complementary)** | 挑選色輪上相對的兩個色相 | 高對比、鮮艷、引起注意 |
| **分裂互補色 (Split-Complementary)** | 基礎色 + 其互補色相鄰的兩個顏色 | 動態但保持平衡 |
| **三等分配色 (Triadic)** | 色輪上等距離的三個色相 | 鮮艷、俏皮、富有創意 |

### 如何選擇配色方案：
1. **專案的氛圍是什麼？** 冷靜 → 相似色。大膽 → 互補色。
2. **需要多少種顏色？** 極簡 → 單色。複雜 → 三等分。
3. **受眾是誰？** 保守 → 單色。年輕 → 三等分。

## 2. 60-30-10 原則

### 分配原則
```
┌─────────────────────────────────────────────────┐
│                                                 │
│     60% 主色 (PRIMARY - 背景、大面積區域)         │
│     → 應為中性或冷靜色                          │
│     → 決定了整體的基調                          │
│                                                 │
├────────────────────────────────────┬────────────┤
│                                    │            │
│   30% 輔色 (SECONDARY)               │ 10% 強調色 │
│   (卡片、區塊、標頭)                 │ (ACCENT)   │
│   → 起輔助作用，不佔主導地位         │ (CTA、高亮)│
│                                    │ → 吸引注意│
└────────────────────────────────────┴────────────┘
```

### 實作模式
```css
:root {
  /* 60% - 根據淺色/深色模式和氛圍挑選 */
  --color-bg: /* 中性色：白色、米白或深灰色 */
  --color-surface: /* 與背景略有不同 */
  
  /* 30% - 根據品牌或情境挑選 */
  --color-secondary: /* 主色的暗淡版本或中性色 */
  
  /* 10% - 根據希望引發的動作/情緒挑選 */
  --color-accent: /* 鮮艷、吸睛 */
}
```

## 3. 色彩心理學 — 意義與選擇

### 如何根據情境選擇

| 如果專案是... | 考慮這些色相 | 原因 |
|------------------|---------------------|-----|
| **金融、科技、醫療** | 藍色、鴨翅綠 (Teals) | 信任、穩定、冷靜 |
| **環保、健康、自然** | 綠色、大地色系 | 成長、健康、有機 |
| **食物、能源、青年** | 橘色、黃色、暖色系 | 食慾、興奮、溫暖 |
| **奢華、美容、創意** | 深鴨翅綠、金色、黑色 | 精緻、進階感 |
| **緊急、銷售、警示** | 紅色、橘色 | 行動、注意、激情 |

### 情感聯想 (用於決策)

| 色相系列 | 正面聯想 | 注意事項 |
|------------|----------------------|----------|
| **藍色** | 信任、冷靜、專業 | 可能顯得冰冷、太過企業化 |
| **綠色** | 成長、自然、成功 | 如果過度使用可能顯得無聊 |
| **紅色** | 激情、急迫、能量 | 高度興奮感，應節制使用 |
| **橘色** | 溫暖、友善、創意 | 若飽和度過高可能顯得廉價 |
| **紫色** | ⚠️ **禁用** — AI 過度使用此色！| 改用深鴨翅綠/栗色/祖母綠代替 |
| **黃色** | 樂觀、注意、快樂 | 難以閱讀，建議作為強調色 |
| **黑色** | 優雅、力量、現代 | 可能顯得沉重 |
| **白色** | 簡潔、極簡、開放 | 可能顯得過於單調/無菌 |

### 選擇過程：
1. **什麼行業？** → 縮小到 2-3 個色相系列
2. **什麼情緒？** → 挑選主色相
3. **什麼對比度？** → 決定淺色或深色模式
4. **詢問使用者** → 在開始前進行確認

## 4. 色票生成原則

### 從單一顏色出發 (HSL 方法)

與其死記十六進位碼，不如學習如何**操作 HSL**：

```
HSL = 色相 (Hue), 飽和度 (Saturation), 明度 (Lightness)

色相 (0-360): 顏色系列
  0/360 = 紅色
  60 = 黃色
  120 = 綠色
  180 = 青色
  240 = 藍色
  300 = 紫色

飽和度 (0-100%): 顏色強度
  低 = 暗淡、精緻
  高 = 鮮艷、充滿活力

明度 (0-100%): 亮度
  0% = 黑色
  50% = 純色
  100% = 白色
```

### 生成完整色譜

給定「任何」基礎色，建立一個比例尺：

```
明度比例 (Lightness Scale):
  50  (最淺)  → L: 97%
  100         → L: 94%
  200         → L: 86%
  300         → L: 74%
  400         → L: 66%
  500 (基礎色) → L: 50-60%
  600         → L: 48%
  700         → L: 38%
  800         → L: 30%
  900 (最深)  → L: 20%
```

### 飽和度調整

| 情境 | 飽和度等級 |
|---------|-----------------|
| **專業/企業** | 較低 (40-60%) |
| **俏皮/青年** | 較高 (70-90%) |
| **深色模式** | 減少 10-20% |
| **無障礙** | 確保對比度，可能需要調整 |

## 5. 基於情境的選擇指南

### 不要複製色票，請遵循以下過程：

**步驟 1：識別情境**
```
什麼類型的專案？
├── 電子商務 → 需要平衡信任感與急迫感
├── SaaS/儀表板 → 需要低疲勞、專注於數據
├── 醫療/健康 → 需要冷靜、自然感
├── 奢華/頂級 → 需要內斂的優雅
├── 創意/作品集 → 需要有個性、令人難忘
└── 其他 → 詢問使用者
```

**步驟 2：選擇主色相系列**
```
根據情境，挑選一個：
- 藍色系 (信任)
- 綠色系 (成長)
- 暖色系 (能量)
- 中性色系 (優雅)
- 或詢問使用者偏好
```

**步驟 3：決定淺色/深色模式**
```
考慮因素：
- 使用者偏好？
- 行業標準？
- 內容類型？ (文字密集 = 偏好淺色)
- 使用時間？ (夜間使用的 App = 需要深色選項)
```

**步驟 4：利用原則生成色票**
- 使用 HSL 操作
- 遵循 60-30-10 原則
- 檢查對比度 (WCAG)
- 使用實際內容進行測試

## 6. 深色模式原則

### 關鍵規則 (無固定色碼)

1. **絕不使用純黑** → 使用帶有輕微色相的極深灰色
2. **絕不使用純白文字** → 使用 87-92% 的明度
3. **降低飽和度** → 鮮艷的顏色在深色模式下會讓眼睛疲勞
4. **高度 = 亮度** → 視覺層級較高的元素應略微亮一點

### 深色模式中的對比度

```
背景層級 (隨著高度增加，由深變淺)：
第 0 層 (底層)   → 最深
第 1 層 (卡片)   → 稍亮
第 2 層 (彈窗)   → 更亮
第 3 層 (浮窗)   → 最亮的深色
```

### 針對深色模式適配顏色

| 淺色模式 | 深色模式調整方式 |
|------------|---------------------|
| 高飽和度強調色 | 降低 10-20% 飽和度 |
| 純白背景 | 帶有品牌色調的深灰色 |
| 黑色文字 | 淺灰色 (非純白) |
| 有色背景 | 降低飽和度、較暗的版本 |

## 7. 無障礙指南 (Accessibility)

### 對比度要求 (WCAG)

| 等級 | 普通文字 | 大型文字 |
|-------|-------------|------------|
| AA (最低) | 4.5:1 | 3:1 |
| AAA (加強) | 7:1 | 4.5:1 |

### 如何檢查對比度

1. **將顏色轉換為亮度 (Luminance)**
2. **計算比例**：(較亮者 + 0.05) / (較暗者 + 0.05)
3. **調整直到比例符合要求**

### 安全模式

| 使用案例 | 指引 |
|----------|-----------|
| **淺色背景上的文字** | 使用 35% 或更低的明度 |
| **深色背景上的文字** | 使用 85% 或更高的明度 |
| **白色上的主色** | 確保變體足夠深 |
| **按鈕** | 背景與文字之間保持高對比 |

## 8. 色彩選擇檢查清單

在確定任何顏色選擇之前，請驗證：

- [ ] **詢問過使用者偏好嗎？** (若未指定)
- [ ] **是否符合專案情境？** (行業、受眾)
- [ ] **是否遵循 60-30-10？** (適當的分配)
- [ ] **是否符合 WCAG？** (已檢查對比度)
- [ ] **在兩種模式下都能運作嗎？** (若需要深色模式)
- [ ] **這不是你的預設/最愛配色吧？** (多樣性檢查)
- [ ] **是否與上一個專案不同？** (避免重複)

## 9. 應避免的錯誤模式 (Anti-Patterns)

### ❌ 不要：
- 每個專案都複製同樣的十六進位碼
- 預設使用紫色/紫羅蘭色 (AI 的傾向)
- 預設使用深色模式 + 霓虹色 (AI 的傾向)
- 使用純黑 (#000000) 背景
- 在深色背景上使用純白 (#FFFFFF) 文字
- 忽略使用者的行業情境
- 跳過詢問使用者偏好的步驟

### ✅ 要：
- 每個專案生成新的色票
- 詢問使用者關於色彩的偏好
- 考慮行業和受眾
- 使用 HSL 進行靈活的操作
- 測試對比度和無障礙性
- 同時提供淺色和深色選項

---

> **記住**：色彩是決策，而非預設。每個專案都值得根據其獨特情境進行深思熟慮的選擇。
