## ⚠️ 關鍵：假設前先詢問（必要）

> **停！如果使用者的請求是開放式的，不要預設你的喜好。**

### 當使用者提示模糊時，詢問：

**顏色未指定？** 詢問：
> 「你偏好什麼調色盤？（藍/綠/橙/中性/其他？）」

**風格未指定？** 詢問：
> 「你想要什麼風格？（極簡/大膽/復古/未來/有機？）」

**佈局未指定？** 詢問：
> 「你有佈局偏好嗎？（單欄/網格/非對稱/全寬？）」

### ⛔ 要避免的預設傾向（反安全港）：

| AI 預設傾向 | 為什麼不好 | 改為思考 |
|-------------|------------|----------|
| **Bento 網格（現代陳腔）** | 每個 AI 設計都用 | 為什麼此內容需要網格？ |
| **Hero 分割（左/右）** | 可預測且無聊 | 巨大排版或垂直敘事如何？ |
| **Mesh/Aurora 漸層** | 「新」的偷懶背景 | 什麼是激進的色彩搭配？ |
| **玻璃態** | AI 的「高級」概念 | 實心、高對比度的平面如何？ |
| **深青色 / 金融科技藍** | 紫色禁令後的安全港 | 為什麼不用紅、黑或霓虹綠？ |

> 🔴 **「你選擇的每個『安全』結構都讓你更接近通用範本。承擔風險。」**

## 1. 約束分析（始終第一步）

任何設計工作前，回答這些或詢問使用者：

| 約束 | 問題 | 為什麼重要 |
|------|------|------------|
| **時程** | 多少時間？| 決定複雜度 |
| **內容** | 已就緒還是佔位？| 影響佈局彈性 |
| **品牌** | 現有指南？| 可能決定色彩/字型 |
| **技術** | 什麼堆疊？| 影響能力 |
| **受眾** | 確切是誰？| 驅動所有視覺決策 |

## 2. UX 心理原則

### 核心定律（內化這些）

| 定律 | 原則 | 應用 |
|------|------|------|
| **希克定律** | 選擇越多 = 決策越慢 | 限制選項、使用漸進式揭露 |
| **費茲定律** | 越大 + 越近 = 越容易點擊 | 適當調整 CTA 大小 |
| **米勒定律** | 工作記憶 ~7 項 | 將內容分組 |
| **馮·雷斯托夫** | 不同 = 記憶深刻 | 讓 CTA 視覺上獨特 |
| **序列位置** | 首/尾最被記住 | 關鍵資訊放在開頭/結尾 |

---

## 3. 佈局原則

### 黃金比例（φ = 1.618）

```
用於比例和諧：
├── 內容 : 側邊欄 = 大約 62% : 38%
├── 每個標題大小 = 上一個 × 1.618
└── 間距可遵循：sm → md → lg（每個 × 1.618）
```

### 8 點網格概念

```
所有間距和大小為 8 的倍數：
├── 緊密：4px（微觀半步）
├── 小：8px
├── 中：16px
├── 大：24px、32px
├── XL：48px、64px、80px
└── 根據內容密度調整
```

## 4. 色彩原則

### 60-30-10 規則

```
60% → 主色/背景（平靜、中性基底）
30% → 次要色（輔助區域）
10% → 強調色（CTA、亮點、注意力）
```

詳細色彩理論：[color-system.md](color-system.md)

---

## 5. 排版原則

### 比例選擇

| 內容類型 | 比例 | 感覺 |
|----------|------|------|
| 密集 UI | 1.125-1.2 | 緊湊、高效 |
| 一般 Web | 1.25 | 平衡（最常見）|
| 編輯 | 1.333 | 可讀、寬敞 |
| Hero/展示 | 1.5-1.618 | 戲劇性影響 |

詳細排版：[typography-system.md](typography-system.md)

## 6. 視覺效果原則

### 陰影層級

```
高度概念：
├── 較高元素 = 較大陰影
├── Y 偏移 > X 偏移（光從上方來）
├── 多層 = 更逼真
└── 暗色模式：可能需要發光代替
```

完整效果指南：[visual-effects.md](visual-effects.md)

## 7. 動畫原則

### 緩動選擇

| 動作 | 緩動 | 原因 |
|------|------|------|
| 進入 | Ease-out | 減速、定位 |
| 離開 | Ease-in | 加速、退出 |
| 強調 | Ease-in-out | 平滑、刻意 |
| 活潑 | Bounce | 有趣、有活力 |

## 9. 反模式

### ❌ AI 傾向模式（避免！）

- **每個專案相同顏色**
- **暗色 + 霓虹作為預設**
- **紫色/紫羅蘭一切（紫色禁令 ✅）**
- **簡單著陸頁用 Bento 網格**
- **Mesh 漸層 & 發光效果**
- **不詢問使用者偏好**
