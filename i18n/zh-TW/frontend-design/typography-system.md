# 排版系統參考指南

> 排版原則與決策 — 學習思考，而非死記硬背。
> **沒有固定的字體名稱或大小 — 理解系統運作。**

---

## 1. 模組化比例原則 (Modular Scale)

### 什麼是模組化比例？

```
字體大小之間的數學關係：
├── 挑選一個基準大小 (BASE size，通常是內文)
├── 挑選一個比例 (RATIO，乘法器)
└── 使用以下公式生成所有大小：base × ratio^n
```

### 常見比例及其使用時機

| 比例名稱 | 數值 | 感受 | 最適合於 |
|-------|-------|---------|----------|
| Minor Second (小二度) | 1.067 | 非常細微 | 密集的 UI、小螢幕 |
| Major Second (大二度) | 1.125 | 細微 | 緊湊的介面 |
| Minor Third (小三度) | 1.2 | 舒適 | 行動 App、卡片 |
| Major Third (大三度) | 1.25 | 平衡 | 一般網頁（最常見）|
| Perfect Fourth (完全四度) | 1.333 | 明顯 | 評論、部落格 |
| Perfect Fifth (完全五度) | 1.5 | 戲劇化 | 標題、行銷頁面 |
| Golden Ratio (黃金比例) | 1.618 | 最大衝擊力 | Hero 區域、大屏顯示 |

### 生成你的比例尺

```
假設：base = 你的基準大小, ratio = 你的比例

比例尺：
├── xs:  base ÷ ratio²
├── sm:  base ÷ ratio
├── base: 你的基準大小 (YOUR_BASE_SIZE)
├── lg:  base × ratio
├── xl:  base × ratio²
├── 2xl: base × ratio³
├── 3xl: base × ratio⁴
└── ... 根據需要繼續
```

### 選擇基準大小

| 情境 | 基準大小範圍 | 原因 |
|---------|-----------------|-----|
| 行動優先 | 16-18px | 小螢幕上的易讀性 |
| 桌面端 App | 14-16px | 資訊密度 |
| 評論/長文 | 18-21px | 長時間閱讀的舒適度 |
| 無障礙重點 | 18px+ | 更容易閱讀 |

## 2. 字體配對原則 (Font Pairing)

### 什麼讓字體能完美搭配

```
對比性 + 和諧感：
├── 差異大到足以建立層級
├── 相似到足以感到凝聚
└── 通常是：襯線體 + 無襯線體，或 標題字 + 中性字
```

### 配對策略

| 策略 | 如何做 | 結果 |
|----------|-----|--------|
| **對比 (Contrast)** | 襯線標題 + 無襯線內文 | 經典、評論/社論感 |
| **同家族 (Same Family)** | 同一種可變字體，不同粗細 | 凝聚、現代 |
| **同設計師 (Same Designer)** | 同一字型設計師的作品 | 通常具有和諧的比例 |
| **年代匹配 (Era Match)** | 來自同一時期的字體 | 歷史一致性 |

### 比較重點

```
配對時，請比較：
├── x-height (小寫字母的高度)
├── 字母寬度 (窄 vs 寬)
├── 線條對比 (粗細變化)
└── 整體氛圍 (正式 vs 隨意)
```

### 安全的配對模式

| 標題風格 | 內文風格 | 氛圍 |
|---------------|------------|------|
| 幾何無襯線 | 人文主義無襯線 | 現代、友善 |
| 展示襯線體 | 簡潔無襯線 | 社論感、精緻 |
| 中性無襯線 | 同一種無襯線 | 極簡、科技感 |
| 粗幾何體 | 細幾何體 | 當代感 |

### 應避免：

- ❌ 兩種裝飾性字體放在一起
- ❌ 過於相似但會產生衝突的字體
- ❌ 超過 2-3 種字體家族
- ❌ x-height 差異巨大的字體

## 3. 行高原則 (Line Height)

### 相互關係

```
行高取決於：
├── 字體大小 (字體越大，所需的行高比例通常越小)
├── 行長 (行越長，需要越多行高)
├── 字體設計 (有些字體天生需要更多空間)
└── 內容類型 (標題 vs 內文)
```

### 不同情境的指引

| 內容類型 | 行高範圍 | 原因 |
|--------------|-------------------|-----|
| **標題** | 1.1 - 1.3 | 行數短，希望緊湊 |
| **內文** | 1.4 - 1.6 | 舒適的閱讀體驗 |
| **長篇文章** | 1.6 - 1.8 | 最大化易讀性 |
| **UI 元素** | 1.2 - 1.4 | 空間效率 |

### 調整因素

- **長行長** → 增加行高
- **大字體大小** → 減少行高比例
- **全大寫 (All caps)** → 可能需要更多行高
- **緊湊字距 (Tracking)** → 可能需要更多行高

## 4. 行長原則 (Line Length)

### 最佳閱讀寬度

```
甜點區：每行 45-75 個字元
├── < 45: 太過破碎，打斷流暢度
├── 45-75: 舒適的閱讀
├── > 75: 眼睛追蹤疲勞
```

### 如何衡量

```css
/* 基於字元的寬度 (推薦) */
max-width: 65ch; /* ch = "0" 字元的寬度 */

/* 這會自動適應字體大小 */
```

### 不同情境的調整

| 情境 | 字元範圍 |
|---------|-----------------|
| 桌面端文章 | 60-75 個字元 |
| 行動裝置 | 35-50 個字元 |
| 側邊欄文字 | 30-45 個字元 |
| 寬螢幕顯示器 | 仍應限制在約 75ch |

## 5. 響應式排版原則 (Responsive Typography)

### 問題所在

```
固定大小無法良好縮放：
├── 桌面端大小在行動裝置上太大
├── 行動端大小在桌面端太小
└── 斷點跳躍感覺不自然
```

### 流體排版 (clamp)

```css
/* 語法：clamp(最小值, 理想值, 最大值) */
font-size: clamp(
  MINIMUM_SIZE,
  FLUID_CALCULATION,
  MAXIMUM_SIZE
);

/* FLUID_CALCULATION 通常是： 
   基準值 + 視窗相關單位 (vw) */
```

### 縮放策略

| 元素 | 縮放行為 |
|---------|-----------------|
| 內文 | 輕微縮放 (1rem → 1.125rem) |
| 次標題 | 中度縮放 |
| 標題 | 戲劇化的縮放 |
| 展示文字 | 最戲劇化的縮放 |

## 6. 粗細與強調原則 (Weight & Emphasis)

### 語意化粗細使用

| 粗細範圍 | 名稱 | 用於 |
|--------------|------|---------|
| 300-400 | Light/Normal | 內文、段落 |
| 500 | Medium | 細微的強調 |
| 600 | Semibold | 次標題、標籤 |
| 700 | Bold | 標題、強力強調 |
| 800-900 | Heavy/Black | 展示文字、Hero 標語 |

### 建立對比

```
好的對比 = 跳過至少 2 個粗細層級
├── 400 內文 + 700 標題 = 優
├── 400 內文 + 500 強調 = 細微
├── 600 標題 + 700 次標題 = 太過相似
```

### 應避免

- ❌ 太多種粗細 (每頁最多 3-4 種)
- ❌ 使用相鄰粗細建立層級 (400/500)
- ❌ 在長篇文章中使用過重的粗細

## 7. 字距原則 (Tracking)

### 原則

```
大文字 (標題)：緊湊的字距
├── 字母很大，間隙顯得更明顯
└── 輕微的負字距看起來更好

小文字 (內文)：標準或略寬的字距
├── 提升小尺寸時的易讀性
└── 內文絕不使用負字距

全大寫 (ALL CAPS)：始終使用較寬的字距
├── 大寫字母缺乏升部/降部
└── 需要更多空間才顯得自然
```

### 調整指引

| 情境 | 字距調整 |
|---------|---------------------|
| 展示文字/Hero | -2% 到 -4% |
| 標題 | -1% 到 -2% |
| 內文 | 0% (標準) |
| 小文字 | +1% 到 +2% |
| 全大寫 (ALL CAPS) | +5% 到 +10% |

## 8. 層級原則 (Hierarchy)

### 透過文字建立視覺層級

```
建立層級的方法：
├── 大小 (SIZE - 最明顯)
├── 粗細 (WEIGHT - 粗體突出)
├── 顏色 (COLOR - 對比程度)
├── 間距 (SPACING - 邊距分隔區塊)
└── 位置 (POSITION - 頂部 = 重要)
```

### 典型的層級結構

| 級別 | 特徵 |
|-------|-----------------|
| 第一級 (H1) | 最大、最粗、最獨特 |
| 第二級 (H2) | 明顯較小但仍為粗體 |
| 第三級 (H3) | 中型大小，可能僅靠粗細區分 |
| 內文 | 標準大小與粗細 |
| 說明文字/註釋 | 更小、通常使用較淡的顏色 |

### 測試層級

詢問自己：「我能一眼看出什麼是最重要的嗎？」

如果你瞇起眼睛看頁面，層級應該依然清晰。

## 9. 易讀性心理學

### F 型閱讀模式

```
使用者以 F 型模式掃描：
├── 沿著頂部 (第一行)
├── 沿著左側向下
├── 再次橫向掃描 (次標題)
└── 繼續沿左側向下
```

**意涵**：將關鍵資訊放在左側和標題中。

### 資訊區塊化以利理解

- 短段落 (最多 3-4 行)
- 清晰的次標題
- 使用清單符號 (Bullet points)
- 區塊之間的留白

### 認知簡易性 (Cognitive Ease)

- 熟悉的字體 = 易於閱讀
- 高對比 = 減少壓力
- 一致的模式 = 可預測

## 10. 排版選擇檢查清單

在確定排版方案之前：

- [ ] **詢問使用者對字體的偏好嗎？**
- [ ] **考慮過品牌/情境嗎？**
- [ ] **選擇了適當的比例比率 (Scale ratio) 嗎？**
- [ ] **是否限制在 2-3 種字體家族內？**
- [ ] **測試過所有比例的易讀性嗎？**
- [ ] **檢查過行長 (45-75ch) 嗎？**
- [ ] **驗證過無障礙的對比度嗎？**
- [ ] **這與你上一個專案不同吧？**

### 應避免的錯誤模式

- ❌ 每個專案都用一樣的字體
- ❌ 使用太過多種字體家族
- ❌ 為了風格而犧牲易讀性
- ❌ 沒有響應式縮放的固定大小
- ❌ 在內文使用裝飾性強的字體

---

> **記住**：排版是關於溝通的清晰度。根據內容需求和受眾進行選擇，而非個人偏好。
