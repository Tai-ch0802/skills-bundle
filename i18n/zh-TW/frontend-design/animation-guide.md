# 動畫指南參考手冊

> 動畫原則與時間心理學 — 學習如何決策，而非複製。
> **不要死記硬背固定的持續時間 — 理解影響時間的因素。**

---

## 1. 持續時間原則 (Duration)

### 影響時間的因素

```
決定動畫速度的因素：
├── 距離 (DISTANCE)：移動距離越遠 = 持續時間越長
├── 尺寸 (SIZE)：元素越大 = 動畫越慢
├── 複雜度 (COMPLEXITY)：複雜內容 = 需要更多處理時間
├── 重要性 (IMPORTANCE)：關鍵動作 = 需要清晰回饋
└── 情境 (CONTEXT)：緊急 = 快，奢華 = 慢
```

### 根據目的劃分的持續時間範圍

| 目的 | 範圍 | 原因 |
|---------|-------|-----|
| 即時回饋 | 50-100ms | 低於感知閾值 |
| 微交互 | 100-200ms | 快速但能被察覺 |
| 標準轉換 | 200-300ms | 舒適的步調 |
| 複雜動畫 | 300-500ms | 留給眼睛跟蹤的時間 |
| 頁面切換 | 400-600ms | 平滑的銜接 |
| **頂級特效/驚喜效果** | 800ms+ | 戲劇化、有機彈簧、分層效果 |

### 選擇持續時間

詢問自己：
1. 元素移動了多遠？
2. 注意到這個變化有多重要？
3. 使用者是在等待，還是在背景運作？

## 2. 緩動原則 (Easing)

### 緩動的作用

```
緩動 = 速度隨時間變化的方式
├── 線性 (Linear)：恆定速度 (感機械、生硬)
├── 緩出 (Ease-out)：啟動快，結束慢 (自然的進入)
├── 緩入 (Ease-in)：啟動慢，結束快 (自然的退出)
└── 緩入緩出 (Ease-in-out)：兩端都慢 (平穩、經過深思熟慮)
```

### 何時使用每種緩動

| 緩動類型 | 最適合於 | 感覺像是 |
|--------|----------|------------|
| **緩出 (Ease-out)** | 進入畫面 | 到達、安頓 |
| **緩入 (Ease-in)** | 離開畫面 | 啟程、退出 |
| **緩入緩出 (Ease-in-out)** | 強調、循環 | 刻意、流暢 |
| **線性 (Linear)** | 持續運動 | 機械、不變 |
| **彈跳/彈性 (Bounce/Elastic)** | 俏皮的 UI | 有趣、精力充沛 |

### 模式範例

```css
/* 進入視圖 = 緩出 (減速) */
.enter {
  animation-timing-function: ease-out;
}

/* 離開視圖 = 緩入 (加速) */
.exit {
  animation-timing-function: ease-in;
}

/* 持續運動 = 緩入緩出 */
.continuous {
  animation-timing-function: ease-in-out;
}
```

## 3. 微交互原則 (Micro-Interactions)

### 什麼是好的微交互

```
微交互的目的：
├── 回饋 (FEEDBACK)：確認動作已發生
├── 引導 (GUIDANCE)：展示可行性
├── 狀態 (STATUS)：指示目前狀態
└── 愉悅 (DELIGHT)：創造微小的快樂時刻
```

### 按鈕狀態

```
懸停 (Hover) → 細微的視覺變化 (抬升、變色、縮放)
點擊 (Active) → 被按下的感覺 (縮小、陰影變化)
焦點 (Focus) → 清晰的指示器 (輪廓、光環)
載入中 (Loading) → 進度指示器 (旋轉圖示、骨架)
成功 (Success) → 確認 (打勾、顏色變化)
```

### 原則

1. **立即反應** (低於 100ms 感知時間)
2. **與動作匹配** (按下 = `scale(0.95)`, 懸停 = `translateY(-4px) + 光暈`)
3. **大膽但流暢** (展現專業感)
4. **保持一致** (相同的動作 = 相同的回饋)

## 4. 載入狀態原則 (Loading States)

### 根據情境劃分的類型

| 情況 | 處理方式 |
|-----------|----------|
| 快速載入 (<1s) | 不需要指示器 |
| 中等 (1-3s) | 旋轉圖示或簡單動畫 |
| 長時間 (3s+) | 進度條或骨架屏 (Skeleton) |
| 未知持續時間 | 不確定進度指示器 |

### 骨架屏 (Skeleton Screens)

```
目的：減少感知等待時間
├── 立即顯示佈局形狀
├── 進行細微動畫 (閃爍、脈動)
├── 準備就緒後替換為真實內容
└── 感覺比旋轉圖示更快
```

### 進度指示器

```
何時顯示進度：
├── 使用者啟動的動作
├── 檔案上傳/下載
├── 多步驟流程
└── 長時間維護的操作

何時「不」需要：
├── 非常快速的操作
├── 背景任務
├── 初始頁面載入 (骨架屏更好)
```

## 5. 頁面切換原則 (Page Transitions)

### 轉換策略

```
簡單規則：快退，慢進
├── 退場內容快速淡出
├── 入場內容動態進入
└── 避免「所有東西同時在動」
```

### 常見模式

| 模式 | 何時使用 |
|---------|-------------|
| **淡入淡出 (Fade)** | 安全預設，適用於任何地方 |
| **滑動 (Slide)** | 序列導航 (上一步/下一步) |
| **縮放 (Scale)** | 開啟/關閉彈窗 |
| **共用元素 (Shared element)**| 保持視覺連續性 |

### 方向匹配

```
導航方向 = 動畫方向
├── 前進 → 從右側滑入
├── 後退 → 從左側滑入
├── 深入 → 從中心放大
├── 退出/回到上層 → 縮小
```

## 6. 捲動動畫原則 (Scroll Animation)

### 漸進式揭露 (Progressive Reveal)

```
內容隨著使用者捲動出現：
├── 減輕初始認知負荷
├── 獎勵探索
├── 絕不能感到笨重延遲
└── 提供關閉選項 (無障礙)
```

### 觸發點 (Trigger Points)

| 觸發時機 | 效果 |
|-----------------|--------|
| 剛進入視窗 | 標準揭露 |
| 在視窗中央 | 用於強調 |
| 部分可見時 | 提早揭露 |
| 完全可見時 | 延後觸發 |

### 動畫屬性

- 淡入 (不透明度 opacity)
- 向上滑動 (變換 transform)
- 縮放 (變換 transform)
- 上述三者的組合

### 效能

- 使用 Intersection Observer
- 僅對 transform/opacity 進行動畫處理
- 在行動裝置上視需要減少效果

## 7. 懸停效果原則 (Hover Effects)

### 效果與動作匹配

| 元素 | 效果 | 意圖 |
|---------|--------|--------|
| **可點擊卡片** | 抬升 + 陰影 | 「這是可以互動的」 |
| **按鈕** | 顏色/亮度變化 | 「按下我」 |
| **圖片** | 縮放/放大 | 「靠近檢視」 |
| **連結** | 底線/變色 | 「導航至此」 |

### 原則

1. **提示互動性** — 懸停顯示它是可點擊的
2. **不要過度** — 細微的變化即見效
3. **重要性匹配** — 變化越大 = 越重要
4. **觸控替代方案** — 懸停在行動裝置上無效

## 8. 回饋動畫原則 (Feedback Animation)

### 成功狀態

```
適當慶祝：
├── 小動作 → 細微打勾/變色
├── 大動作 → 更顯著的動畫
├── 完成 → 令人滿足的成就感動畫
└── 匹配品牌個性
```

### 錯誤狀態

```
吸引注意力而不引發恐慌：
├── 顏色變化 (語義紅)
├── 抖動動畫 (簡短！)
├── 聚焦於錯誤欄位
└── 清晰的訊息告知
```

### 時間控制

- 成功：時間略長 (享受這一刻)
- 錯誤：快速 (不要拖慢處理動作)
- 載入：持續直到完成

## 9. 效能原則

### 動畫開銷低的屬性

```
GPU 加速 (快速)：
├── transform: translate, scale, rotate
└── opacity: 0 到 1

CPU 密集 (慢速)：
├── width, height
├── top, left, right, bottom
├── margin, padding
├── 圓角變化 (border-radius)
└── 陰影變化 (box-shadow)
```

### 優化策略

1. 儘可能**僅對 transform/opacity** 進行動畫
2. **避免佈局觸發** (尺寸/位置變化)
3. **節制使用 will-change** (給瀏覽器的提示)
4. **在低階裝置上測試** (而不僅僅是開發機器)

### 尊重使用者偏好

```css
@media (prefers-reduced-motion: reduce) {
  /* 尊重此偏好 */
  /* 僅保留必要的動畫 */
  /* 減少或移除裝飾性動效 */
}
```

## 10. 動畫決策檢查清單

在添加動畫之前：

- [ ] **它有目的嗎？** (回饋/引導/愉悅)
- [ ] **時間是否恰當？** (不會太快/太慢)
- [ ] **挑選了正確的緩動嗎？** (進入/退出/強調)
- [ ] **效能優良嗎？** (僅 transform/opacity)
- [ ] **測試過減弱動態嗎？** (無障礙)
- [ ] **與其他動畫一致嗎？** (相同的節奏感)
- [ ] **這不是你的預設值吧？** (多樣性檢查)
- [ ] **若不明確，詢問過使用者風格嗎？**

### 應避免的錯誤模式

- ❌ 每個專案都用相同的時間參數
- ❌ 為了動畫而動畫
- ❌ 忽略減弱動態偏好
- ❌ 對開銷大的屬性進行動畫
- ❌ 同時有太多東西在動
- ❌ 讓使用者感到沮喪的延遲

---

> **記住**：動畫就是溝通。每個動效都應該具有意義，並為使用者體驗服務。
