# SEO 基礎

> 搜尋引擎可見性原則。

---

## 1. E-E-A-T 框架

| 原則 | 信號 |
|------|------|
| **經驗（Experience）** | 第一手知識、真實範例 |
| **專業（Expertise）** | 資歷、知識深度 |
| **權威性（Authoritativeness）** | 反向連結、提及、產業認可 |
| **可信度（Trustworthiness）** | HTTPS、透明度、準確資訊 |

---

## 2. Core Web Vitals

| 指標 | 目標 | 測量項目 |
|------|------|----------|
| **LCP** | < 2.5s | 載入效能 |
| **INP** | < 200ms | 互動性 |
| **CLS** | < 0.1 | 視覺穩定性 |

---

## 3. 技術 SEO 原則

### 網站結構

| 元素 | 用途 |
|------|------|
| XML sitemap | 協助爬取 |
| robots.txt | 控制存取 |
| Canonical 標籤 | 防止重複 |
| HTTPS | 安全信號 |

### 效能

| 因素 | 影響 |
|------|------|
| 頁面速度 | Core Web Vital |
| 行動友善 | 排名因素 |
| 乾淨 URL | 可爬取性 |

---

## 4. 內容 SEO 原則

### 頁面元素

| 元素 | 最佳實踐 |
|------|----------|
| Title 標籤 | 50-60 字元、關鍵字在前 |
| Meta description | 150-160 字元、引人注目 |
| H1 | 每頁一個、主要關鍵字 |
| H2-H6 | 邏輯層級 |
| Alt 文字 | 描述性、不堆砌 |

### 內容品質

| 因素 | 重要性 |
|------|--------|
| 深度 | 全面覆蓋 |
| 新鮮度 | 定期更新 |
| 獨特性 | 原創價值 |
| 可讀性 | 清晰寫作 |

---

## 5. Schema 標記類型

| 類型 | 用途 |
|------|------|
| Article | 部落格文章、新聞 |
| Organization | 公司資訊 |
| Person | 作者個人資料 |
| FAQPage | Q&A 內容 |
| Product | 電商 |
| Review | 評分 |
| BreadcrumbList | 導航 |

---

## 6. AI 內容指南

### Google 尋找什麼

| ✅ 要 | ❌ 不要 |
|-------|---------|
| AI 草稿 + 人工編輯 | 發布未編輯的 AI 內容 |
| 新增原創見解 | 不加價值地複製 |
| 專家審查 | 跳過事實查核 |
| 遵循 E-E-A-T | 關鍵字堆砌 |

---

## 7. 排名因素（依優先順序）

| 優先 | 因素 |
|------|------|
| 1 | 高品質、相關內容 |
| 2 | 權威網站的反向連結 |
| 3 | 頁面體驗（Core Web Vitals）|
| 4 | 行動最佳化 |
| 5 | 技術 SEO 基礎 |

---

## 8. 測量

| 指標 | 工具 |
|------|------|
| 排名 | Search Console、Ahrefs |
| 流量 | Analytics |
| Core Web Vitals | PageSpeed Insights |
| 索引 | Search Console |
| 反向連結 | Ahrefs、Semrush |
